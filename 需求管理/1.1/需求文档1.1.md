## 一、项目概述

### 1.1 系统定位

**DailyReview（周目标管理系统）** 是一套面向多部门团队的企业级目标管理平台，覆盖**周目标制定 → 日任务分解 → 进度跟踪 → 数据统计 → 自动通知**的完整闭环。

### 1.2 技术架构

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| **前端** | Vue 3.5 + Vite 6.2 + Element Plus 2.9 | SPA，Hash 路由，Composition API |
| **后端** | PHP + PDO | RESTful API，无框架 |
| **数据库** | MySQL（InnoDB） | 10 张业务表 |
| **图表** | ECharts 5.6 + vue-echarts 7.0 | 统计可视化 |
| **外部集成** | OA 系统、钉钉机器人 | 任务同步 & 群消息推送 |
| **部署** | `https://daily.gameyzy.com` | 线上环境 |

### 1.3 核心用户角色

| 角色 | 操作范围 |
|------|---------|
| 部门管理者 | 创建/编辑/删除周目标，查看统计，发送群通知 |
| 执行人 | 查看分配任务，更新日任务进度 |
| 测试人员 | 管理测试任务，导入 Excel 测试计划 |
| 系统管理员 | 部门/用户管理，系统设置 |

### 3.3 WeekGoalAPI.php — 周目标管理

| Action | 方法 | 功能 | 核心参数 |
|--------|------|------|---------|
| `list` | GET | 按多周查询周目标 | `mondayDates`(逗号分隔), `department_id`, `country` |
| `get` | GET | 按单周+多条件查询 | `mondayDate`, `department_id`, `executor`, `status`, `priority`, `country` |
| `getWeekPeriod` | GET | 获取最近 20 个周周期 | 无 |
| `batch_create` | POST | 批量创建周目标 | 数组格式目标数据 |
| `create` | POST | 单个创建周目标 | 目标完整字段 |
| `update` | POST | 更新周目标 | `id` + 可更新字段 |
| `delete` | POST | 删除周目标 + 关联日目标 | `id` |

**业务规则：**
- 创建周目标时自动创建日目标（daily_goals 表），多执行人用 `/` 分隔并逐一创建
- 更新周目标时同步更新日目标：删除不再关联的执行人，新增新执行人
- 删除周目标时级联删除关联日目标
- 使用事务保证数据一致性
- 使用字段白名单限制可更新字段

### 3.7 TestTask.php — 测试任务管理

| Action | 功能 | 说明 |
|--------|------|------|
| `create` | 创建/更新测试任务 | MD5(负责人+测试内容) 去重 |
| `query` | 查询测试任务 | 多条件筛选 + 时间范围 |

**去重机制：**
1. 计算 `md5key = MD5(responsible_person + test_content)`
2. 查询 test_tasks 表是否存在该 md5key
3. 存在 → 获取已有 task_id；不存在 → 插入新记录
4. 检查 test_tasks_info 是否有相同日期记录，避免重复

**同步流程：**

### 4.1 数据库表结构总览

系统共 10 张业务表，数据库名 `dailyreview`，字符集 UTF-8。

### 4.2 表结构详细定义

#### 4.2.1 weekly_goals — 周目标表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PK, AUTO_INCREMENT | 主键 |
| department_id | INT | NOT NULL | 所属部门 |
| executor | VARCHAR(250) | NOT NULL | 执行人名称（多人用 `/` 分隔） |
| executor_id | TEXT | NOT NULL | 执行人 ID（多人用 `/` 分隔） |
| weekly_goal | TEXT | NOT NULL | 目标内容 |
| priority | INT | NOT NULL | 优先级：10=S, 9=A, 8=B, 7=C |
| status | INT | DEFAULT 0 | 状态：0=草稿, 1=进行中, 2=测试中, 3=已上线, 4=已暂停, 5=已完成 |
| process | DECIMAL(10,2) | DEFAULT 0.00 | 进度百分比 |
| remark | TEXT | NOT NULL | 备注 |
| cross_week | INT | DEFAULT 0 | 是否跨周：0=否, 1=是 |
| is_new_goal | TINYINT(1) | DEFAULT 0 | 是否新需求 |
| createdate | INT | NOT NULL | 创建日期 YYYYMMDD |
| mondayDate | INT | NOT NULL | 所属周的周一日期 |
| pre_finish_date | INT | NULL | 预计完成日期 |
| real_finish_date | INT | NULL | 实际完成日期 |
| country | VARCHAR(20) | DEFAULT '' | 地区代码 |
| version | VARCHAR(20) | DEFAULT '' | 版本号 |

**索引：**
- `weekly_goals_mondayDate_index`：(mondayDate, real_finish_date, department_id)
- `weekly_goals_version_index`：(version)

---

| pre_finish_date | INT | NULL | 预计完成日期 |

### 4.3 表关系图

```
departments (1) ──────────── (N) users
     │                              │
     │                              │
     ├──── (N) weekly_goals ────────┤ (executor_id 以 / 分隔存储)
     │            │                 │
     │            │ 1:N             │
     │            ▼                 │
     ├──── (N) daily_goals         │
     │                              │
     ├──── (N) daily_tasks ─────────┘ (executor_id → users.id)
     │
     ├──── (N) daily_tasks_today
     │
     └──── (N) today_target

test_tasks (1) ──── (N) test_tasks_info (task_id → test_tasks.id)

watch_user (独立) ── executor_id 指向 users.id（无外键约束）
```

## 五、外部系统集成

| 项目 | 说明 |
|------|------|
