# 跟进事项汇报 — 产品需求文档

> 版本：v1.0  
> 日期：2026-03-15  
> 需求来源：CEO每日汇报模板

---

## 一、需求概述

基于 CEO 每日汇报模板，开发「跟进事项汇报」功能模块。支持每日填报、查看历史记录、按日期筛选，格式覆盖业务事项进展、AI事项进展、跨部门协调事项、风险与异常预警四大板块。

---

## 二、数据库设计（MySQL）

### 2.1 主表：`follow_up_reports`（跟进汇报主表）

```sql
CREATE TABLE `follow_up_reports` (
  `id` int(11) NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `reporter` varchar(50) NOT NULL COMMENT '汇报人',
  `report_date` int(8) NOT NULL COMMENT '汇报日期 YYYYMMDD',
  `week_day` varchar(10) DEFAULT NULL COMMENT '星期几（周一~周日）',
  `coordination_matters` text DEFAULT NULL COMMENT '跨部门协调事项（JSON数组或纯文本）',
  `status` tinyint(1) NOT NULL DEFAULT 0 COMMENT '状态 0=草稿 1=已提交',
  `created_at` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_reporter_date` (`reporter`, `report_date`),
  KEY `idx_report_date` (`report_date`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='跟进事项汇报主表';
```

### 2.2 明细表：`follow_up_report_items`（汇报事项明细）

```sql
CREATE TABLE `follow_up_report_items` (
  `id` int(11) NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `report_id` int(11) NOT NULL COMMENT '关联主表ID',
  `category` varchar(30) NOT NULL COMMENT '分类：business=业务事项, ai=AI事项, risk=风险预警',
  `section_title` varchar(100) NOT NULL COMMENT '模块标题（如：支付平台、S级需求、AI全员使用）',
  `responsible_person` varchar(100) DEFAULT NULL COMMENT '负责人（如：韩益忠 + 钱贵祥）',
  `section_order` int(3) NOT NULL DEFAULT 0 COMMENT '排序序号',
  `content` text NOT NULL COMMENT '事项内容（JSON格式存储多个字段）',
  `risk_level` tinyint(1) NOT NULL DEFAULT 0 COMMENT '风险等级 0=无 1=黄色关注 2=红色紧急',
  `created_at` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  KEY `idx_report_id` (`report_id`),
  KEY `idx_category` (`category`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='汇报事项明细表';
```

### 2.3 content 字段 JSON 结构说明

不同 `category` 的 `content` 存储格式：

**业务事项（category=business）：**
```json
// 支付平台
{
  "progress": "已完成支付渠道对接",
  "risk_or_blocker": "无"
}

// S级需求
{
  "dev_progress": "陈苏熙已完成核心开发",
  "test_progress": "王雪斌测试中",
  "has_b_level_mixed": "否"

// 投放攻坚
{
  "attack_progress": "解决了投放回调延迟问题",
  "delivery_data": "当日消耗 12W，ROI 1.5"

// 产品核心指标
{
  "fission": "裂变率 15%",
  "new_pay_rate": "新增付费率 8.2%",
  "first_to_second_pay": "一付到二付 32%"
}
```

**AI事项（category=ai）：**
```json
// AI全员使用
{
  "management_plan": "已提交：张三、李四；未提交：王五",
  "game_tech_drill": "第3天，已完成基础模型训练"

// AI工具库

  "new_tools": "新增：自动化报表生成工具",
  "issue_summary": "本日收集3个问题，已解决2个"

```

**风险预警（category=risk）：**
```json
{
  "description": "服务器负载过高，需扩容",
  "risk_level": 2
}
```

## 三、后端 API 设计

### 3.1 接口文件：`FollowUpReportAPI.php`

| 操作 | action 值 | 请求方式 | 参数 | 说明 |
|------|----------|---------|------|------|
| 获取汇报列表 | `list` | GET | `report_date`, `reporter`（可选） | 按日期查询汇报列表 |
| 获取汇报详情 | `get` | GET | `id` 或 `report_date` + `reporter` | 获取含明细的完整汇报 |
| 创建/保存汇报 | `save` | POST | 主表+明细数据（JSON body） | 新增或更新（upsert） |
| 提交汇报 | `submit` | POST | `id` | 将草稿改为已提交 |
| 删除汇报 | `delete` | POST | `id` | 删除汇报及关联明细 |
| 复制历史汇报 | `copy` | POST | `source_date`, `target_date`, `reporter` | 复制某天模板到新日期 |
| 导出 Markdown | `export_md` | GET | `id` 或 `report_date` + `reporter` | 导出为 Markdown 文本 |

## 四、前端需求设计

### 4.1 路由配置

在 `router/index.js` 中新增路由：

```javascript
{
  path: 'follow-up-report',
  name: 'FollowUpReport',
  component: () => import('@/view/FollowUpReport.vue')
}
```

侧边栏菜单新增入口：「跟进事项汇报」

### 4.2 页面结构：`FollowUpReport.vue`

#### 顶部工具栏
| 控件 | 说明 |
|------|------|
| 日期选择器 | 默认当天日期，YYYYMMDD 格式 |
| 汇报人下拉框 | 默认当前登录用户，可切换查看他人汇报 |
| 查询按钮 | 按日期+汇报人查询 |
| 复制昨日按钮 | 复制前一天的汇报作为今日模板 |
| 提交按钮 | 将草稿状态改为已提交 |
| 导出MD按钮 | 导出为 Markdown 格式文本，支持复制到剪贴板 |

#### 表单区域 — 四大板块

**板块一：今日业务事项进展**

| 模块 | 负责人 | 填写项 | 控件类型 |
|------|--------|--------|---------|
| 支付平台 | 韩益忠 + 钱贵祥 | 进展、风险/卡点 | textarea × 2 |
| S级需求 | 陈苏熙 + 王雪斌 | 开发进度、测试进度、是否有B级任务混入 | textarea × 2 + select |
| 投放攻坚 | 梁浩风 + 张梁 | 攻坚进展、投放数据 | textarea × 2 |
| 产品核心指标 | 张梁 | 裂变、新增付费率、一付到二付 | textarea × 3 |

**板块二：今日AI事项进展**

| 模块 | 填写项 | 控件类型 |
|------|--------|---------|
| AI全员使用 | 管理层方案收集、游戏技术组AI演练 | textarea × 2 |
| AI工具库 | 新增工具、使用问题汇总 | textarea × 2 |

**板块三：跨部门协调事项**

| 填写项 | 控件类型 |
|--------|---------|
| 需CEO关注或决策的事项 | textarea（支持多行） |

**板块四：风险与异常预警**

| 等级 | 填写项 | 控件类型 |
|------|--------|---------|
| 🔴 紧急事项 | 事项描述 | textarea + 动态增减行 |
| 🟡 需关注事项 | 事项描述 | textarea + 动态增减行 |

#### Markdown 预览区

- 右侧或下方实时预览最终 Markdown 输出效果
- 展示格式与原始模板完全一致
- 支持一键复制 Markdown 文本

### 4.3 交互逻辑

| 场景 | 逻辑 |
|------|------|
| 进入页面 | 自动加载当天汇报，若不存在则展示空模板 |
| 自动保存 | 编辑内容变更后 3 秒自动保存为草稿（防抖） |
| 复制昨日 | 调用 `copy` 接口，将前一天内容复制并填充到表单 |
| 提交 | 校验必填项（至少填写一个板块），调用 `submit` 接口 |
| 导出MD | 将表单数据拼装为 Markdown 模板格式，写入剪贴板 |
| 查看历史 | 切换日期后自动加载对应日期的汇报（只读或可编辑取决于状态） |
| 移动端适配 | 四大板块纵向堆叠，预览区域折叠 |

### 4.4 UI 组件使用（Element Plus）

```
el-date-picker       — 日期选择
el-select             — 汇报人选择
el-button             — 操作按钮
el-card               — 四大板块容器
el-form / el-form-item — 表单包装
el-input type="textarea" — 文本输入
el-tag                — 风险等级标签（red/warning）
el-collapse           — 移动端板块折叠
el-message            — 操作反馈提示
el-divider            — 板块分隔
```

## 七、非功能需求

- 自动保存防抖时间：3 秒
- API 超时：5 秒（复用现有 http.js 配置）
- 每人每天仅一条汇报记录（UNIQUE 约束）
- 已提交的汇报默认不可再编辑（可查看）
- 移动端响应式适配
